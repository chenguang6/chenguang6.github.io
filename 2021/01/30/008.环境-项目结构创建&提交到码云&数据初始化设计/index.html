环境-项目结构创建&提交到码云

从gitee初始化一个项目

idea开发工具导入项目

拉取代码说明

  • URL：为git项目的地址

  • Directory为我们项目存储本机地址

创建微服务项目

  • 总项目右键 ——> New ——> Module
  • 使用Spring微服务向导 ——> Spring Initializr

创建商品服务

  • Group ： com.light.lightmall
  • Artifact ：lightmall-product
  • Language ：java
  • Java Version ：8
  • Description ： 谷粒商城-商品服务
  • Package：com.light.lightmall.product
  • 导入组件： SpringWeb 、OpenFeign

微服务模块

  • 商品服务（product）
  • 仓储服务（ware）
  • 订单服务（order）
  • 优惠券服务（coupon）
  • 用户服务（member）

lightmall-project 设置为总项目

  1. 创建聚合工程的pom.xml

    忽略不必要的文件，不提交到版本控制中

  2. ​ 在总项目的.gitignore文件进行添加，进行忽略没必要提交的代码

    代码提交到git

  3. idea安装gitee插件，Settings ——> Plugins ——> 安装gitee插件

  4. 下方菜单栏Version Control ——> Default Changelist ——> 右键 ——> commit(提交到本地仓库) ——> Commit And Push

初始化数据库

数据库路径

  • 路径：https://pan.baidu.com/s/1l9NE2eNu-TqAvB_xcS_94w
  • 密码：j85b

依次创建数据库，将sql导入

  • lightmall_oms (订单系统)
  • lightmall_pms (商品系统)
  • lightmall_sms (营销系统)
  • lightmall_ums (用户系统)
  • lightmall_wms (库存系统)
  • utf8mb4能够兼容utf8并能解决一些字符乱码问题
  • 复制sql内容，再mysql中执行，如果直接执行sql可能会存在数据编码问题
